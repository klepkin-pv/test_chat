# 💬 Chat Real - Чат в реальном времени

Современное чат-приложение с WebSocket соединениями, комнатами, отправкой файлов, end-to-end шифрованием и статусами присутствия.

## 🚀 Технологический стек

### Frontend
- **Next.js 15** - React фреймворк с App Router и Turbopack
- **React 19** - Современная библиотека для UI
- **TypeScript** - Типизированный JavaScript
- **Tailwind CSS 4** - Utility-first CSS фреймворк (без отдельного конфига!)
- **Socket.io Client** - WebSocket клиент
- **Zustand** - Легковесное управление состоянием
- **React Hook Form** - Управление формами

### Backend
- **Node.js** - JavaScript runtime
- **Express.js** - Web фреймворк
- **Socket.io** - WebSocket сервер
- **TypeScript** - Типизированный JavaScript
- **MongoDB (latest)** - NoSQL база данных
- **Mongoose** - ODM для MongoDB
- **Redis (latest)** - In-memory хранилище для кеширования
- **JWT** - Аутентификация
- **bcryptjs** - Хеширование паролей

### DevOps
- **Docker** - Контейнеризация
- **Docker Compose** - Оркестрация контейнеров

## 🛠 Установка и запуск

### Локальная разработка (рекомендуется)

1. **Установите зависимости:**
   ```bash
   cd backend && npm install
   cd ../frontend && npm install
   ```

2. **Настройте .env файл** (уже настроен для локального запуска)

3. **Запустите сервисы:**
   ```bash
   # Terminal 1 - Backend
   cd backend && npm run dev
   
   # Terminal 2 - Frontend  
   cd frontend && npm run dev
   ```

Приложение доступно: **http://localhost:5176/chat**

### Docker (для продакшена)

```bash
cp .env.docker .env
docker-compose up --build -d
```

Приложение доступно: **http://localhost:5175/chat**

> **Важно:** Исправлены API роуты (убрано дублирование `/api/`) и PostCSS конфигурация для совместимости с Next.js 15

## 🌐 Доступ к сервисам

- **Frontend (Next.js):** http://localhost:5175/chat
- **Backend API:** http://localhost:4000
- **MongoDB:** localhost:27018
- **Redis:** localhost:6380

## 🔧 Решение конфликта портов

Если у вас уже запущен проект на localhost:5175, есть два варианта:

### Вариант 1: Запуск в Docker (рекомендуется)
```bash
docker-compose up --build -d
```
Приложение будет доступно на http://localhost:5175/chat

### Вариант 2: Nginx Reverse Proxy
Настройте Nginx для проксирования `/chat` на этот проект:
```nginx
location /chat {
    proxy_pass http://localhost:5175/chat;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
}
```

## 👤 Система администрирования

После первого запуска создай супер-админа:

```bash
cd backend
npm run create-admin
```

Логин: `admin` / Пароль: `123123` (защищен от изменений)

### Роли:
- **User** - обычный пользователь
- **Moderator** - может банить в комнатах
- **Admin** - может управлять ролями, банить глобально, создавать комнаты

### Баны:
- **Room ban** - бан в одной комнате (пользователь видит комнату, но не может писать)
- **Global ban** - бан везде, включая личные сообщения
- Временные (1-999 часов) или постоянные (0 часов)

Клик на имя пользователя открывает карточку управления (для модераторов и админов).
